    컴퓨터가 음수를 취하는 방법¶

    컴퓨터는 0/1 만 안다. - 기호 없다. 그럼 -5 같은 음수는 어떻게 표현하나? 정답: 2의 보수 (Two's Complement).

    현대 모든 CPU (x86, ARM, RISC-V) 가 사용하는 표준 음수 표현법.

    1. 왜 필요한가¶

    자연스러운 발상: - "맨 앞 비트를 부호로 쓰자!" → 부호 비트 (sign-magnitude) - "비트 다 뒤집자!" → 1의 보수 - "뒤집고 +1 하자!" → 2의 보수 ✅ 채택

    세 가지 방식 비교 (8bit, -5 표현):

    방식 -5 비트 문제
    부호 비트 1000 0101 +0(0000 0000)과 -0(1000 0000) 둘 다 존재. 덧셈 회로 복잡
    1의 보수 1111 1010 +0/-0 여전히 둘
    2의 보수 1111 1011 0 하나, 덧셈/뺄셈 회로 통합 가능

    2. 2의 보수 만드는 법¶

    양수 → 음수 변환 2단계:

    1) 모든 비트 뒤집기 (NOT) 2) +1

    예시: +5 → -5 (8 bit)¶

    +5 = 0000 0101 ↓ NOT 1111 1010 ← 1의 보수 ↓ +1 -5 = 1111 1011 ← 2의 보수

    역변환: 음수 비트 → 절댓값¶

    같은 절차 한 번 더 적용.

    -5 = 1111 1011 ↓ NOT 0000 0100 ↓ +1 0000 0101 = 5 → 부호 붙여 -5 확인

    3. MSB = 부호 비트¶

    최상위 비트 (MSB) 1 = 음수, 0 = 양수.

    8bit 의미
    0000 0000 0
    0000 0001 1
    0111 1111 127 (양수 최대)
    1000 0000 -128 (음수 최소)
    1111 1110 -2
    1111 1111 -1

    -1 은 왜 전부 1?¶

    ``` 0 - 1 = ?

    0000 0000 - 0000 0001 ───────────── 1111 1111 ← borrow 발생, underflow → -1 ```

    기억: 0xFF = -1 (signed byte), 0xFFFFFFFF = -1 (signed int).

    4. 범위 공식¶

    n bit 2의 보수 범위:

    -2^(n-1) ~ +2^(n-1) - 1

    Java 타입 bit 범위
    byte 8 -128 ~ 127
    short 16 -32,768 ~ 32,767
    int 32 -2,147,483,648 ~ 2,147,483,647 (≈ ±21억)
    long 64 -2⁶³ ~ 2⁶³-1 (≈ ±9.2 × 10¹⁸)

    비대칭 — 음수가 1개 더 많음. 0이 양수 쪽에 들어가서.

    5. 왜 2의 보수가 천재적인가¶

    덧셈 회로 하나로 뺄셈도 처리.

    5 + (-3) 계산 (8 bit):

    +5 = 0000 0101 -3 = 1111 1101 (3 = 0000 0011 → NOT → 1111 1100 → +1 → 1111 1101) ───────────── 0000 0010 ← +2 ✅ (carry 버림) (1)

    carry out (9번째 비트) 자동 버림. 결과 정확.

    → CPU 입장: 뺄셈 = 부호 바꿔서 덧셈. 별도 뺄셈 회로 불필요. 회로 단순 = 칩 작고 빠름.

    이게 부호 비트/1의 보수 대신 2의 보수가 표준이 된 결정적 이유.

    6. 빠른 변환 트릭¶

    NOT + 1 매번 하지 않고:

    오른쪽부터 첫 1까지 그대로, 그 왼쪽 비트 전부 뒤집기.

    +12 = 0000 1100 ↑ 첫 1 (오른쪽부터 보면 여기) → -12 = 1111 0100 뒤집 그대로

    익숙해지면 즉시 변환 가능.

    7. 부호 확장 (Sign Extension)¶

    작은 타입 → 큰 타입으로 옮길 때 MSB 를 위쪽 비트에 복사.

    byte b = -1; // 1111 1111 int i = b; // 1111 1111 1111 1111 1111 1111 1111 1111 = -1 // ↑ MSB 1 을 위쪽 24 비트에 복사

    unsigned 처럼 보려면 마스크:

    java int unsigned = b & 0xFF; // 0000 0000 ... 1111 1111 = 255

    8. Java 에서 확인¶

    ```java byte b = -1; System.out.printf("%02x%n", b & 0xFF); // ff

    byte b2 = -128; System.out.printf("%02x%n", b2 & 0xFF); // 80

    int i = -1; System.out.printf("%08x%n", i); // ffffffff

    System.out.println(Integer.MIN_VALUE); // -2147483648 System.out.println(Integer.MAX_VALUE); // 2147483647 System.out.println(Integer.MAX_VALUE + 1); // -2147483648 (overflow → wrap around) ```

    Overflow 주의¶

    java byte b = 127; b = (byte)(b + 1); // -128 // 0111 1111 + 1 = 1000 0000 = -128

    CPU 는 그냥 비트 연산만. overflow 검사 안 함 — 플래그만 세움.

    9. 자주 보는 음수 비트 패턴 (8bit)¶

    비트 10진 hex
    1111 1111 -1 0xFF
    1111 1110 -2 0xFE
    1111 1100 -4 0xFC
    1111 1000 -8 0xF8
    1111 0000 -16 0xF0
    1110 0000 -32 0xE0
    1100 0000 -64 0xC0
    1000 0000 -128 0x80

    규칙: 절댓값이 2의 거듭제곱이면, 양수 비트 패턴의 1을 왼쪽으로 밀고 그 자리부터 왼쪽 모두 1로 채운 모양.

    10. 흔한 오해¶

    ❌ "1000 = 16 이니까 -16 은 1111 일 것"¶

    틀림. - 1000 (4bit) = 8 (16 아님!). 16 = 10000 (5bit). - 1111 (4bit) = -1 (4bit signed) 또는 15 (unsigned). - 8bit -16 의 정답: 1111 0000 = 0xF0.

    ❌ "MSB가 1이면 절댓값을 그냥 음수로 읽으면 된다"¶

    틀림. 부호 비트 방식과 혼동. - 1000 0001 = 부호 비트 방식이면 -1, 2의 보수 방식이면 -127. - 현대 CPU = 2의 보수만 사용. 항상 NOT + 1 로 해석.

    11. 핵심 요약¶

    1. 양수 → 음수: NOT + 1
    2. MSB = 부호 비트 (1=음수)
    3. n bit 범위: -2^(n-1) ~ 2^(n-1) - 1
    4. -1 = 전부 1 (0xFF...F)
    5. 덧셈 회로 하나로 뺄셈 처리 → CPU 단순
    6. Java byte 는 signed. unsigned 보려면 & 0xFF
    7. 자릿값(2⁰=1, 2¹=2, ..., 2⁷=128) 외워야 헤매지 않음

    연습문제¶

    Q1. 8bit 에서 -7 의 2진수?

    답

    7 = 0000 0111 → NOT → 1111 1000 → +1 → 1111 1001 = 0xF9

    Q2. 1000 1010 (8bit signed) 의 10진 값?

    답

    MSB=1 → 음수. NOT: 0111 0101 +1: 0111 0110 = 64+32+16+4+2 = 118 → -118

    Q3. 16bit 에서 -256 은 hex 로?

    답

    256 = 0x0100 → NOT → 0xFEFF → +1 → 0xFF00

    Q4. Java byte b = (byte) 200; 의 값은?

    답

    200 = 1100 1000. byte 는 signed → MSB 1 → 음수. NOT: 0011 0111, +1: 0011 1000 = 56 → -56

    Q5. 32bit int 의 -1 을 hex 로?

    답

    0xFFFFFFFF (32개 비트 전부 1)

    Q6. -16 을 8bit 2진수로?

    답

    16 = 0001 0000 → NOT → 1110 1111 → +1 → 1111 0000 = 0xF0
